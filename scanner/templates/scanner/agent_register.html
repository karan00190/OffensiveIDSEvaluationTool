What happens next
1
Server generates a unique auth_token and secret_key for this agent.
2
You copy both into agent_config.json on your lab endpoint machine.
3
Run python agent.py on the endpoint. It connects to this server and starts sending heartbeats.
4
Every request from the agent is signed with an HMAC signature. The server verifies it before accepting any data.
Security model
Every request carries three security layers:

Authorization: Bearer <token>
→ Identifies which agent is calling

X-Timestamp: <unix time>
→ Prevents replay attacks (5 min window)

X-Signature: <HMAC-SHA256>
→ Proves the body was not tampered with
The secret_key is shown exactly once after registration. It is not stored in plaintext on the server. If lost, you must delete and re-register the agent.
